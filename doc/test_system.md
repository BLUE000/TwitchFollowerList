# 総合テスト仕様書 (System Test Specification)

要件定義に基づき、実際のユーザーの操作フローに沿ってアプリケーション全体の振る舞いを検証します。

## 1. テスト方針
- ビルドされた実行ファイル（`.exe`）を使用し、実際のマウス操作およびキーボード操作による手動テスト（E2Eテスト）を行う。
- 実際のTwitchアカウントとネットワーク環境を使用し、本番同等の環境で動作を確認する。

## 2. 総合テストケース

### ST-01: 初回ログインと自動取得
- **手順**:
  1. ツールを起動する。
  2. 「Twitch Login」ボタンをクリックする。
  3. ブラウザが開き、Twitchで「許可」を押す。
  4. ツールに戻る。
- **期待結果**:
  - ボタンが「ログイン済み」になり操作不可になる。
  - 画面右側のリストビューに、自分のチャンネルのフォロワー一覧が表示される。
  - `out/AllList.dat` が作成されており、ツールとは別のデコーダ（確認用スクリプト等）でデコードすると正しいCSVになっている。

### ST-02: フォルダ管理機能とドラッグ＆ドロップ
- **手順**:
  1. 「フォルダ作成」ボタンで「VIP」グループを作成する。
  2. リストビューからフォロワーを1名、「VIP」フォルダにドラッグ＆ドロップする。
- **期待結果**:
  - ツリービューに「VIP」が表示される。
  - ファイルシステム上に `out/VIP/Lists.dat` と `out/GroupsList.dat` が作成される。
  - 「VIP」フォルダをクリックすると、リストビューに先ほどドロップした1名のみが表示される。
  - ツリービューのルート（すべて）をクリックすると、全員が再表示される。

### ST-03: Undo / Redo 機能と履歴の永続化
- **手順**:
  1. ST-02の直後、「Undo」ボタンをクリックする。
- **期待結果**:
  - フォロワーが「VIP」フォルダから除外され、全表示時のグループIDも空（未所属）に戻る。
  - `out/VIP/Lists.dat` から該当ユーザーが消える。
- **手順**:
  2. アプリケーションを一度終了し、再度起動する。
  3. 「Redo」ボタンをクリックする。
- **期待結果**:
  - アプリ再起動後も `Config/ActionHistory.dat` から履歴が復元され、UI上で直近の操作としてRedoが成功する。
  - フォロワーが再び「VIP」グループに所属した状態になる。
### ST-04: ニックネーム・メモの編集と永続化
- **手順**:
  1. リストビューの「ニックネーム」列または「メモ」列をダブルクリックし、任意の文字列を入力して確定する。
  2. アプリケーションを終了し、再度起動する。
- **期待結果**:
  - 再起動後も、入力したニックネームおよびメモの内容がリスト上に正しく表示されていること。
  - 暗号化された `.dat` ファイルを復号して確認した場合、入力した内容が正しく記録されていること。
