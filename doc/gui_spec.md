# GUI仕様 (GUI Specifications)

## 1. 画面レイアウト
メインウィンドウは、主に「上部のコントロールエリア」と「下部のデータ表示エリア」に分かれる。
下部のデータ表示エリアは、QSplitterを用いて左右に分割される。

### 1.1. 上部コントロールエリア
- **Twitchログインボタン (`twitchLoginButton`)**
  - 役割: クリックすることでTwitch OAuth認証フローを開始する。
  - 状態遷移: 認証完了時に有効なトークンを保持している場合は非活性化（Disabled）になる。
- **アンドゥ / リドゥ ボタン**
  - 役割: フォルダ管理やアサイン操作の取り消し（Undo）とやり直し（Redo）を行う（UI上は直近5回まで）。

### 1.2. 下部データ表示エリア（左右分割）
- **左ペイン: ツリービュー (`outDirTreeView`)**
  - 役割: 作成されたグループ（フォルダ）を階層構造で表示する。
  - 構成: 最上位に「ルート（すべて）」ノードがあり、その下に各グループフォルダや「未所属」フォルダが並ぶ。
- **右ペイン: リストビュー (`followerListView`)**
  - 役割: ツリービューで選択されたフォルダ（グループ）に所属するフォロワー一覧を絞り込み表示する。ルートを選択した場合は全フォロワーを表示する。
  - 操作: リスト内のアイテムをツリービューのフォルダへドラッグ＆ドロップし、グループへ所属させる。

## 2. イベントとアクション
- **Twitchログインボタン押下**
  - ブラウザ等を開いてTwitchのOAuth認証ページへリダイレクト。
  - 認証成功後、トークンを受け取ってボタンを非活性化する。
- **（※今後の追加仕様）**
  - リストビューのアイテムクリック時のアクション
  - ツリービューでのファイル選択時のプレビュー/アクション 等
