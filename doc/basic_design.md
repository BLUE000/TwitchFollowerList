# 基本設計 (Basic Design)

## 1. アーキテクチャ思想
役割ごとにディレクトリ（レイヤー）を分割し、関心の分離を行う。
1. **UI層 (`ui/`)**: 画面の描画やユーザー入力を担当する。
2. **API層 (`api/`)**: 外部との通信（Twitch API、OAuth認証）や、ローカルファイルへの入出力を担当する。
3. **Core層 (`core/`)**: UIとAPIの間に立ち、ビジネスロジックやデータ加工を担うコントローラー層。

## 2. モジュール構成
1. **UIモジュール (`ui/`)**
   - Qt Designerファイル (.ui) およびそれを制御するViewクラス。
2. **Coreモジュール (`core/`)**
   - アプリケーションのメインロジック（Controller / Manager）。
3. **APIモジュール (`api/`)**
   - **認証 (Auth)**: Twitch OAuth認証フローの実行。
   - **通信 (Network)**: Twitch APIへのリクエスト、パース。
   - **ファイル操作 (IO)**: 保存データの暗号化・復号、ファイルの読み書き。

## 3. データ連携フロー
1. 起動時、`core` が `api` にトークン検証を依頼。
2. ログイン完了時、`core` が `api` を通じてフォロワー一覧を取得。
3. **データ保存**: `api` 内の `FileManager` が、**`TransCipher` ライブラリ** を呼び出してデータを暗号化し、ローカルに保存する。

## 4. ディレクトリ構成
```text
FollowerList/
├── doc/            # ドキュメント類
├── out/            # 出力データ配置用ディレクトリ
├── ui/             # UI層
├── core/           # 内部処理層
└── api/            # 外部連携層
```

## 5. 外部ライブラリ
- **TransCipher (DLL)**: 
  - データの秘匿性を確保するための共通暗号化コンポーネント。
  - `api/FileManager` がこの DLL をリンクし、動的レシピ生成による暗号化を適用する。
